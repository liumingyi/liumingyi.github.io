Android屏幕适配

尺寸单位：

参考Android官方文档

参考2

px (pixel): 像素。 （设计师给的设计图通常都是以此为单位。）

dpi (dots per inch): 像素密度，每英寸的像素点数。

dp (density-independent pixel): 同时也叫做dip，密度无关像素，与终端上的实际物理像素点无关。密度无关像素等于 160 dpi 屏幕上的一个物理像素。

注意哦：dp即dip，dip不是dpi，切勿混淆，为了不混淆还是忘掉dip吧，它就是dp！

另外物理尺寸：1英寸 = 2.54 cm

平时所说的手机屏幕尺寸是指的对角线长度。而 “像素密度 = 对角线像素 / 对角线的物理尺寸”。

非要弄个公式的话: $$屏幕像素密度(dpi) = \frac{\sqrt{长(px)^2 + 宽(px)^2}}{尺寸(inch)}$$

比如：Samsung s7 edge手机，屏幕尺寸5.5英寸，分辨率2560 x 1440像素。

$$s7的像素密度 = \frac{\sqrt{2560^2 + 1440^2}}{5.5} \approx 534 $$

为简便起见，Android 将所有屏幕密度分组为六种通用密度： 低、中、高、超高、超超高和超超超高

密度 屏幕像素密度
ldpi（低） ~120dpi
mdpi（中） ~160dpi
hdpi（高） ~240dpi
xhdpi（超高） ~320dpi
xxhdpi（超超高） ~480dpi
xxxhdpi（超超超高） ~640dpi

通用尺寸：

尺寸 最小尺寸（dp）
小屏幕 426 x 320
正常屏幕 470 x 320
大屏幕 640 x 480
超大屏幕 960 x 720

配置限定符

通用尺寸的限定符从 Android 3.2（Api 13）开始，已经弃用，就忘了这个吧

Android 3.2以后的配置限定符：

屏幕配置 限定符 说明
smallestWidth sw< N > dp 示例：sw600dp sw720dp 屏幕的基本尺寸，由可用屏幕区域的最小尺寸指定。 具体来说，设备的 smallestWidth 是屏幕可用高度和宽度的最小尺寸（您也可以将其视为屏幕的“最小可能宽度”）。无论屏幕的当前方向如何，您均可使用此限定符确保应用 UI 的可用宽度至少为 N dp。smallestWidth 是设备的固定屏幕尺寸特性；设备的 smallestWidth 不会随屏幕方向的变化而改变。
可用屏幕宽度 w< N >dp 示例：w720dp w1024dp 指定资源应该使用的最小可用宽度（dp 单位） — 由 N 值定义。当屏幕的方向在横屏与竖屏之间切换时，系统对应的 宽度值将会变化，以 反映 UI 可用的当前实际宽度。
可用屏幕高度 h< N >dp 示例：h720dp h1024dp 指定资源应该使用的最小屏幕高度（dp 单位） — 由 N 值定义。当屏幕的方向在横屏与竖屏之间切换时，系统 对应的高度值将会变化，以 反映 UI 可用的当前实际高度。

如何适配

  1. 不必说，当然是使用 dp ，而不是像素为单位，忘掉AbsoluteLayout（很久很久以前就已被官方弃用）
  2. 多用RelativeLayout，LinearLayout，ConstraintLayout，善用wrap_content,match_parent,weight来控制视图组件的宽度和高度。
  3. 使用限定符，根据屏幕的配置来加载相应的UI布局。
  4. 配置不同分辨率的图片资源，使用自动拉伸的.9图，使图片资源自适应屏幕尺寸。
